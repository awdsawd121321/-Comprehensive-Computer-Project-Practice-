# FunASR 模型使用与性能测试指南

## 📚 关于 FunASR 模型

### 是否需要训练？

**答案：不需要！可以直接使用预训练模型。**

你的项目使用的是阿里达摩院开源的 FunASR 预训练模型：

- **paraformer-zh** (~944MB) - 中文语音识别模型
- **fsmn-vad** - 语音活动检测模型
- **ct-punc** - 标点符号预测模型

这些模型已经在大规模中文语音数据上训练完成，可以直接用于：
- ✅ 普通话识别
- ✅ 老年人语音识别
- ✅ 日常对话场景
- ✅ 用药提醒、健康咨询等场景

### 什么时候需要训练/微调？

只有在以下情况才需要考虑训练：

| 场景 | 是否需要训练 | 说明 |
|------|------------|------|
| 普通话日常对话 | ❌ 不需要 | 预训练模型已足够 |
| 老年人语音 | ❌ 不需要 | 预训练模型支持 |
| 特定方言（粤语、四川话） | ✅ 需要 | 需要方言数据微调 |
| 专业术语（医学、法律） | ✅ 需要 | 需要领域数据微调 |
| WER > 10% 且无法优化 | ✅ 考虑 | 可能需要微调 |

**对于你的项目（颐伴老年虚拟数字人陪护助手），预训练模型完全够用！**

## 🚀 快速开始

### 1. 启动 FunASR 服务

```bash
# 方式 1: 使用 npm 脚本（推荐）
npm run dev:tts

# 方式 2: 直接运行脚本
./scripts/start_funasr.sh

# 方式 3: 手动运行
conda activate emobit
python scripts/funasr_server.py
```

**首次启动说明：**
- 会自动下载模型（约 500MB）
- 模型加载需要 1-3 分钟
- 看到 "✅ 模型加载完成" 后即可使用

### 2. 运行性能测试

```bash
# 完整性能测试（推荐）
npm run funasr:test

# 或直接运行
node scripts/test-funasr-performance.mjs
```

## 📊 性能测试内容

性能测试工具会自动执行以下测试：

### 测试 1: 连接性测试
- 测试 WebSocket 连接是否正常
- 测量连接建立时间

### 测试 2: 基础识别流程测试
- 测试完整的识别流程
- 测量 Ready 时延、识别时延、总时延

### 测试 3: 响应时延测试（10轮）
- 多次测试取平均值
- 计算统计数据：平均、最小、最大、P50、P95

### 测试 4: 并发性能测试（5并发）
- 测试并发识别能力
- 计算成功率和平均耗时

### 测试 5: 稳定性测试（可选）
- 长时间运行测试
- 评估服务稳定性

## 📈 性能指标说明

### 时延指标

| 指标 | 说明 | 优秀标准 | 良好标准 |
|------|------|---------|---------|
| **Ready 时延** | 从连接到服务就绪的时间 | < 200ms | < 500ms |
| **识别时延** | 从发送音频到返回结果的时间 | < 1000ms | < 2000ms |
| **总时延** | 完整识别流程的总时间 | < 1000ms | < 3000ms |

### 并发性能

| 指标 | 说明 | 优秀标准 | 良好标准 |
|------|------|---------|---------|
| **并发成功率** | 并发请求的成功比例 | ≥ 95% | ≥ 90% |
| **平均耗时** | 并发情况下的平均响应时间 | < 2000ms | < 4000ms |

## 🔧 性能优化建议

### 1. 使用 GPU 加速

如果有 NVIDIA GPU，可以启用 GPU 加速：

```python
# 修改 scripts/funasr_server.py
model = AutoModel(
    model="paraformer-zh",
    vad_model="fsmn-vad",
    punc_model="ct-punc",
    device="cuda",  # 改为 cuda
    disable_update=True,
)
```

**性能提升：** 识别速度提升 2-3 倍

### 2. 调整音频参数

```python
# 减少音频块大小，降低时延
MAX_CHUNK_SIZE = 160000  # 约5秒（默认10秒）
```

**效果：** 降低时延，但可能影响长句识别准确率

### 3. 启用中间结果

```python
# 启用实时识别反馈
ENABLE_INTERIM_RESULTS = True
```

**效果：** 用户可以看到实时识别结果，体验更好

### 4. 增加服务器资源

- **CPU**: 建议 4 核以上
- **内存**: 建议 8GB 以上
- **网络**: 本地部署，延迟最低

## 🐛 常见问题

### Q1: 模型加载失败

**原因：** 依赖未安装或版本不兼容

**解决：**
```bash
pip install "numpy<2" funasr modelscope websockets
```

### Q2: 连接超时

**原因：** 服务未启动或端口被占用

**解决：**
```bash
# 检查服务是否运行
lsof -i :10095

# 重启服务
npm run dev:tts
```

### Q3: 识别时延过高

**原因：** CPU 性能不足或音频数据过大

**解决：**
- 使用 GPU 加速
- 减少音频块大小
- 关闭其他占用 CPU 的程序

### Q4: 并发成功率低

**原因：** 服务器资源不足

**解决：**
- 增加内存
- 减少并发数
- 使用负载均衡

## 🎓 技术细节

### 模型架构

- **Paraformer**: 非自回归端到端语音识别模型
- **FSMN-VAD**: 基于 FSMN 的语音活动检测
- **CT-Punc**: 基于 Conformer 的标点符号预测

### 模型特点

- ✅ 端到端识别，无需额外的语言模型
- ✅ 支持长音频识别（自动分块）
- ✅ 自动添加标点符号
- ✅ 支持实时流式识别
- ✅ 中文识别准确率高

### 性能特点

- **准确率**: WER 通常在 5-10% 之间（普通话）
- **速度**: CPU 模式约 0.3-0.5x 实时（1秒音频需要 2-3秒处理）
- **内存**: 约 2-3GB（模型加载后）
- **并发**: 支持多路并发识别
